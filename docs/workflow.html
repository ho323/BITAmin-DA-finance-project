Analysis Workflow

2025년 상반기 분석 기준으로, 데이터 적재부터 후보 종목 확인과 K-FI 검증까지 이어지는 재현 가능한 순서입니다.

1

DB 초기화

schema, yearly partition, index, ETL 로그 테이블을 준비합니다.

2

주식/지수 백필

2025-01-02부터 2025-06-30까지 전 종목과 시장지수를 적재합니다.

3

ETF 기준일 수집

ETF 일봉, NAV/괴리율, PDF holdings를 기준일 단위로 수집합니다.

4

노출 테이블 생성

종목별 ETF 보유 주식수, 평가금액, 편입 ETF 수를 CSV로 확인합니다.

5

K-FI 산출

ownership, liquidity, leveraged/inverse, deviation, flow stress를 z-score로 결합합니다.

6

이벤트 검증

초과 하락률, decile test, 회귀분석을 대시보드와 CSV 리포트로 검토합니다.

primary key 기준 upsert로 같은 날짜를 다시 실행해도 덮어쓰기됩니다. fact table은 날짜 기준 yearly range partition을 사용합니다.

K-FI Calculation

K-FI는 ETF가 특정 종목에 만드는 보유 압력과 유동성 충격 가능성을 중심에 두고, 한국 ETF 시장 특성인 레버리지/인버스와 괴리율 스트레스를 더합니다.

Base

K-FI Base

ETF 보유와 유동성 압력을 50:50으로 결합합니다.

0.50 * z(ownership_pressure)
+ 0.50 * z(liquidity_pressure)
Korea

K-FI Korea

한국 ETF 시장의 구조적 스트레스를 반영한 MVP 산식입니다.

0.30 * z(ownership_pressure)
+ 0.30 * z(liquidity_pressure)
+ 0.15 * z(leveraged_inverse_pressure)
+ 0.15 * z(deviation_stress)
+ 0.10 * z(flow_stress)
Exposure

Core Components

종목별 ETF holding을 stock universe와 결합합니다.

ownership = ETF held shares / listed shares proxy
liquidity = ETF valuation / avg trading value 20d
Validation

Event Study

K-FI가 높은 종목이 이벤트일에 더 크게 초과 하락하는지 검증합니다.

excess_drop = -(stock_return - market_return)
model = excess_drop ~ KFI + controls

Operational Commands

팀원이 같은 환경에서 다시 실행할 때 필요한 대표 명령입니다. KRX live 수집에는 KRX_ID, KRX_PW가 필요합니다.

로컬 서비스와 DB 초기화 
docker compose up --build
.venv/bin/bitamin-finance init-db
2025년 상반기 주식/지수 백필 
.venv/bin/bitamin-finance backfill \
  --start-date 2025-01-02 \
  --end-date 2025-06-30 \
  --target stock

.venv/bin/bitamin-finance backfill \
  --start-date 2025-01-02 \
  --end-date 2025-06-30 \
  --target market-index
ETF 기준일 수집 
.venv/bin/bitamin-finance ingest \
  --date 2025-06-30 \
  --target etf

.venv/bin/bitamin-finance ingest \
  --date 2025-06-30 \
  --target etf \
  --max-etfs 20
종목별 ETF 편입 현황 CSV 
.venv/bin/bitamin-finance export-exposure \
  --date 2025-06-30 \
  --from-db \
  --output-dir data/processed/exposure_20250630
Airflow: http://localhost:8080 Streamlit: http://localhost:8501 PostgreSQL: localhost:5432 Smoke test: --max-etfs

Operating Notes

프로젝트를 이어받는 사람이 먼저 확인해야 할 제약과 안전장치입니다.

날짜 범위

2025-01-01은 휴장일이고 2025-06-31은 존재하지 않으므로, H1 분석은 2025-01-02부터 2025-06-30까지 사용합니다.

재실행 정책

fact table은 primary key 기준 upsert입니다. 수집이 중간에 멈추면 같은 날짜 명령을 다시 실행합니다.
